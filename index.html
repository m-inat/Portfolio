Site Git

訓練校のテストでPHPを利用したフォーム画面を制作する課題があり、
私自身が通う訓練校のアンケートページを制作しました。
PHPで取得した値を今回はメールで送信されるようにしました。

制作期間：
5日間
担当行程：
情報設計・デザイン・テキスト・コーディング・アップロード
使用言語：
HTML5・CSS3・jQuery・PHP
使用アプリ：
Photoshop・XD・Atom

課題と解決

問題点1：
通常のチェックボックスやラジオボタンだとスマホの場合に押しづらい
問題点2：
テスト時の初期設定でバリデートをPHPで行い、送信ボタンをタップしエラーがあった場合は違うページにとびエラーを表示させ、元のページに戻って入力し直してもらう設定にしていた（無駄な工数が多い）
問題点3：
全ての入力必須項目を入力してもらいたい／離脱を防ぎたい
解決策1：
チェックボックスとラジオボタンを大きなボタンにし、
スマホの場合はPCの時よりボタンを大きくする。
解決策2：
バリデートは基本的にHTMLのrequired属性を使う。
必須項目を入力・チェックすると「次へ」「送信」ボタンを押せるようにする。
解決策3：
入力した際に多少インタラクティブな動きを付け、能動的に入力している感覚を抱いてもらう。
選択肢は極限まで減らし、必要な項目のみ必須とすることでユーザーの負担を減らす。
